# 少样本思维链 Skill

## 任务

根据用户提供的问题、可选相关问题的推理过程、最终答案和输出格式，生成一个使用思维链（Chain-of-Thought, CoT）技术的 Prompt。默认目标是“撰写提示词”，不是直接完成用户问题；只有当用户明确要求执行生成后的 Prompt 时，才继续生成最终答案。

少样本思维链的关键不是只给 `输入 -> 输出`，而是给出 `问题/输入 -> 推理过程 -> 最终答案` 的示例三元组，让模型模仿示例中的分步推理方式，再解决新的目标问题。若用户没有提供完整推理案例，本 skill 不追问，自动退回零样本思维链：让模型自行进行简明逐步推理，并说明原因。

## 输入要求

- 用户任务：说明希望模型解决哪类问题，如数学应用题、常识推理、符号推理、日期计算、代码逻辑分析等。
- 目标问题：用户希望最终处理的真实问题，或 `{用户输入的问题}` 占位符。
- 可选相关问题示例：用户可以在第一次请求中直接提供 1~5 组与目标任务同类型的示例。
- 少样本思维链案例推荐包含：问题/输入、推理过程、最终答案。
- 输出格式：说明最终 Prompt 应要求模型输出哪些部分，如“推理过程 + 最终答案”、JSON、选项字母、代码块等。
- 可选信息：角色设定、约束条件、是否隐藏详细推理、语言要求、答案格式。

示例不是必需项。若用户已经附上完整推理案例，直接吸收案例中的分步推理方式和答案格式；若未检测到完整推理案例，不要追问或中断流程，按零样本思维链生成，并在生成结果中简短说明“未检测到完整用户推理案例，因此按零样本思维链提示生成”。

## 流程概览

1. 读取用户任务、目标问题、可选推理示例和输出格式。
2. 明确角色、任务指令、少样本思维链示例、输出要求和待处理问题。
3. 将完整用户示例整理为 `问题/输入 -> 推理过程 -> 最终答案`，插入生成的 Prompt；如果没有完整推理示例，写入零样本思维链说明。
4. 输出”生成的少样本思维链 Prompt”。
5. 返回最终 Prompt 给用户；不要默认继续执行 Prompt。

## 论文要点

根据《Chain-of-Thought Prompting Elicits Reasoning in Large Language Models》的方法，少样本思维链通过在提示词中提供带有自然语言中间推理步骤的示例，帮助模型处理多步推理任务。论文中的核心启发：

- 示例应是三元组：输入、思维链推理过程、最终输出。
- 推理过程应位于最终答案之前，让模型先分解问题再回答。
- 自然语言推理步骤比只给公式或只增加 token 更有效。
- 该方法更适合需要多步推理的任务，如算术推理、常识推理、符号操作和状态追踪。
- 简单单步任务不一定需要少样本思维链，可使用零样本提示或少样本提示。

## 纯 Skill 可选案例规则

用户可以在第一次请求中直接附上案例；skill 直接读取并注入 Prompt 模板。推荐输入结构：

```json
{
  "task": "解决小学数学应用题",
  "role": "你是一名严谨的数学解题助手",
  "examples": [
    {
      "input": "小明有 5 支铅笔，又买了 2 盒，每盒 3 支。他现在有几支铅笔？",
      "reasoning": "小明原来有 5 支铅笔。2 盒铅笔每盒 3 支，所以新增 2 × 3 = 6 支。总数是 5 + 6 = 11 支。",
      "output": "11"
    }
  ],
  "target_input": "{用户输入的数学应用题}",
  "output_format": "先输出推理过程，再输出最终答案。",
  "constraints": ["推理步骤要简洁", "最终答案单独成行"]
}
```

执行规则：

1. `examples` 可省略；省略时按零样本思维链生成，并在示例区说明原因。
2. 如果提供 `examples`，校验数量为 1~5，且每组都有非空 `input`、`reasoning` 和 `output`。
3. 将完整示例格式化为 `示例 N / 问题 / 推理过程 / 最终答案`，插入生成的 Prompt。
4. 示例仅用于本轮 Prompt 生成，不自动持久化。
5. 如果用户只要求生成提示词，输出 Prompt 即止；如果用户要求继续运行，再使用生成的 Prompt 处理目标问题。

## 注意事项

- 示例如果存在，必须与目标问题同类型，并展示可复用的推理模式。
- 推理示例不要只写“因为显然如此”，要写出关键中间步骤。
- 示例数量优先 2~3 个；任务复杂或格式多样时可用 4~5 个。
- 输出格式要区分推理过程和最终答案，避免模型把答案埋在推理中。
- 对选择题，最终答案应明确选项字母和选项内容。
- 对数学、日期、状态追踪等任务，推理中要保留关键计算或状态变化。
- 如果用户不希望暴露完整推理，可在生成的 Prompt 中要求输出“简要依据 + 最终答案”，但示例仍应体现推理结构。
